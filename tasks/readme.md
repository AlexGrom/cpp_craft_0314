Руководство по работе с CMake в рамках нашего курса.

Данное руководство позволяет создать проект для вашей любимой среды разработки. 

Для того, чтобы создать проект с решением нужно совершить "ритуальный танец", который включает следующие шаги:

1. Найти шаблон проекта, который находится в solutions\ihar_marozau\< номер домашней работы >. 
   Например, solutions\ihar_marozau\1 для 1-ой домашней работы. 

2. Скопировать шаблон проекта в ВАШУ рабочую директорию /solutions/< user_name >.

3. В папке sources шаблонного проекта создать директорию с решением <номер_задачи>, где "номер_задачи" - соответствует идентификатору задачи в задании, 
   например 1_1, 1_2 или 1_3 для задач из 1-ой домашней работы. Это будет подпроект с решением.

4. В директории с подпроектом создать файл CMakeLists.txt со следующим содержанием: compile_project( ${module_name} "*.cpp" "*.h" BINARY binaries ). 
   Пример можно взять из lections\1\code_examples\cmake_example\sources\functions\
   !!!Важно!!! Содержание этого файла может изменяться от домашней работы к домашней работе. Мы будет обращать внимание на изменения. Будьте внимательны

5. Добавить исходные файлы в папку с подпроектом. 

6. Создать CMake файл-генератор для соответствующей среды разработки. За основу можно взять _msvc_gen_2005_64.bat или _gcc_gen.sh. 
   В строчке запуска cmake генератора заменить имя среды разработки. Информацию по именам можно найти на сайте cmake.org или погуглить.
   Например, для Visual Studio 2010 переименовываем файл _msvc_gen_2005_64.bat в _msvc_gen_2010_64.bat и заменяем строчку  
   cmake -DVERBOSE=OFF -DCMAKE_BUILD_TYPE=%BUILD_TYPE% -DSOLUTION_NAME=%SOLUTION_NAME%  -G "Visual Studio 8 2005 Win64" ../    
   на
   cmake -DVERBOSE=OFF -DCMAKE_BUILD_TYPE=%BUILD_TYPE% -DSOLUTION_NAME=%SOLUTION_NAME%  -G "Visual Studio 10 Win64" ../   

7. Запустить файл-генератор по умолчанию создает Debug проект. Если вы хотите сгенерировать Release проект, 
   то вам нужно вызвать генератор с аргументом "Release" ( например для Visual Studio 2005: _msvc_gen_2005_64.bat Release )
   
8. Проект для вашей среды разработки можно найти в директории _build_Debug_64 для Debug решения и в _build_Release_64 для Release.

9. Готово! Успехов! 

Скомпилированные исполняемые файлы находятся в папке < ваш cmake проект >\_build_Debug_64\bin_64\Debug для Debug решения соответственно. 

!!!Важно!!! Не добавляйте файлы через контекстное меню вашей среды разработки. Для добавления исходных файлов в проект 
            нужно в ручную создавать или копировать их в папку с подпроектом, после запускать CMake генератор.
			
!!!Важно!!! Не забывайте добавлять ваши папки и исходные файлы в git (git add). Пустые папки добавить нельзя.
